# 要件定義書

## アプリ概要

アイデア出しに特化したSNS風Webメモアプリ。フォルダ（コルクボード）の中に付箋を自由配置し、ユーザー同士でいいね・コメントを通じてアイデアを深められる。

- **アプリ名**: Memo App
- **技術スタック**: Python / Flask / SQLAlchemy / SQLite / Jinja2 / Bulma CSS / JavaScript
- **起動ポート**: 8888

---

## 画面構成

```
/                       ← フォルダ一覧画面（index.html）
/folder/<id>            ← コルクボード画面（board.html）付箋を自由配置・描画
/login                  ← ログイン画面（login.html）
/register               ← アカウント登録画面（register.html）
/logout                 ← ログアウト（POST）
```

## データモデル

### User

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `username` | Text | NOT NULL, UNIQUE | ログインID |
| `password_hash` | Text | NOT NULL | ハッシュ化されたパスワード |

### Folder

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `name` | Text | NOT NULL | フォルダ名 |
| `user_id` | Integer | FOREIGN KEY → User.id | 作成者 |

### MemoItem

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `title` | Text | NOT NULL | 付箋のタイトル |
| `body` | Text | NOT NULL | 付箋の本文（Markdown） |
| `color` | Text | NOT NULL, default=`#fef08a` | 付箋の背景色（HEX） |
| `tags` | Text | nullable | カンマ区切りのタグ文字列（例: `アイデア,課題`） |
| `pos_x` | Integer | NOT NULL, default=40 | コルクボード上のX座標 |
| `pos_y` | Integer | NOT NULL, default=40 | コルクボード上のY座標 |
| `is_public` | Boolean | NOT NULL, default=False | 公開フラグ |
| `updated_at` | DateTime | NOT NULL | 更新日時（自動） |
| `user_id` | Integer | FOREIGN KEY → User.id | 作成者 |
| `folder_id` | Integer | FOREIGN KEY → Folder.id | 所属フォルダ |

### Vote（新規追加）

付箋へのいいね。1ユーザーが1付箋に1票のみ投票可能。

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `memo_id` | Integer | FOREIGN KEY → MemoItem.id | 対象の付箋（CASCADE削除） |
| `user_id` | Integer | FOREIGN KEY → User.id | 投票者 |
| `created_at` | DateTime | NOT NULL | 投票日時 |

- `(memo_id, user_id)` に UNIQUE 制約（二重投票防止）

### Comment（新規追加）

付箋に紐づくコメント。

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `memo_id` | Integer | FOREIGN KEY → MemoItem.id | 対象の付箋（CASCADE削除） |
| `user_id` | Integer | FOREIGN KEY → User.id | 投稿者 |
| `body` | Text | NOT NULL | コメント本文 |
| `created_at` | DateTime | NOT NULL | 投稿日時 |

### Drawing

| カラム | 型 | 制約 | 説明 |
|--------|----|------|------|
| `id` | Integer | PRIMARY KEY | 自動採番 |
| `folder_id` | Integer | FOREIGN KEY → Folder.id | 所属フォルダ（CASCADE削除） |
| `user_id` | Integer | FOREIGN KEY → User.id | 作成者 |
| `type` | Text | NOT NULL | 描画種別: `pen` / `arrow` |
| `data` | Text | NOT NULL | 描画データ（JSON文字列） |
| `color` | Text | NOT NULL, default=`#e74c3c` | 線の色（HEX） |
| `width` | Integer | NOT NULL, default=3 | 線の太さ（px） |
| `created_at` | DateTime | NOT NULL | 作成日時（自動） |

- データベースファイル: `memo.sqlite`（初回起動時に自動生成）

## 機能要件

### 1. アカウント管理 ✅（実装済み）

- ログインID＋パスワードで新規登録・ログインができる
- パスワードは `werkzeug.security` でハッシュ化して保存する
- 未ログイン時は保護されたページへのアクセスをログイン画面へリダイレクトする
- ログアウトでセッションを破棄する
- スコープ外: パスワード再設定・変更

### 2. フォルダ管理 ✅（実装済み）

#### フォルダ一覧画面（`/`）
- **全ユーザーのフォルダ**を一覧表示する
- 他ユーザーが作成したフォルダにはオーナー名を表示する
- 「＋ 新規フォルダ」で新規作成できる
- フォルダをクリックするとコルクボード画面へ遷移する
- フォルダ削除は**作成者のみ**可能（削除ボタンは作成者にのみ表示）
- フォルダ削除時、中の付箋・描画データもすべて削除される（CASCADE）

### 3. コルクボード（付箋の自由配置） ✅（実装済み）

- **全ログインユーザー**がどのフォルダにもアクセス・付箋追加できる
- 付箋は**タイトル＋本文プレビュー（Markdownレンダリング）**をカード上に表示する
- 付箋をドラッグ&ドロップで移動でき、位置はDBに保存する
- コルクボード画面内でキーワード検索（タイトル・本文の部分一致）ができる

### 4. 付箋（メモ）の編集 ✅（実装済み）

- 付箋をクリックするとモーダルが開き、タイトル・本文を編集できる
- 本文はMarkdown対応。付箋カード上でプレビュー表示される
- モーダルに「保存」「削除」「公開/非公開切り替え」ボタンをまとめて表示する
- タイトルは空欄不可

### 5. 公開/非公開 ✅（実装済み）

| | 公開メモ | 非公開メモ |
|---|---|---|
| 表示 | 全ログインユーザーに見える | 作成者のみ |
| 編集・削除 | 全ログインユーザーが可能 | 作成者のみ |
| 公開/非公開の切り替え | 作成者のみ | 作成者のみ |

### 6. マークダウン対応 ✅（実装済み）

- 付箋の本文欄でMarkdown記法が使用できる
- 付箋カード上にMarkdownプレビューを表示する（高さ制限あり）
- ライブラリ: `marked.js`（CDN）

### 7. ダークモード ✅（実装済み）

- ヘッダーにダークモード切替ボタンを設ける
- 切替状態はブラウザの `localStorage` に保存する

### 8. 描画ツール ✅（実装済み）

- ペン / 矢印 / 消しゴム / 移動モードを切り替えできる
- 色選択・線の太さ（3段階）・Undo・全消去
- 付箋カードは描画レイヤーより常に手前に表示する

### 9. タグ＆カラーラベル（新規追加）

付箋にタグと背景色を設定して視覚的に分類できる。

#### タグ

- 付箋にカンマ区切りで複数のタグを設定できる（例: `アイデア, 課題, 決定事項`）
- コルクボード画面でタグ一覧を表示し、クリックで該当タグの付箋のみ絞り込み表示できる
- 付箋カード上にタグをバッジ表示する

#### カラーラベル

- 付箋の背景色をプリセット色から選択できる（黄 / 緑 / 青 / ピンク / 白 の5色）
- 新規作成・編集モーダルで色を選択できる
- 付箋カードの背景色に反映する

### 10. いいね（投票）（新規追加）

- 公開付箋に「👍 いいね」ボタンを表示する
- 1ユーザーが1付箋に1回のみいいね可能（再クリックで取り消し）
- いいね数を付箋カード上に表示する
- 自分がいいね済みかどうかをボタンの見た目で区別する（色・塗りつぶし）
- いいね操作はAJAX（fetch API）で即時反映し、ページリロードは不要
- 非公開付箋にはいいねボタンを表示しない

#### APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/memo/<id>/vote` | いいねをトグル（追加/取り消し） |

### 11. コメント（新規追加）

- 付箋のモーダル内にコメント欄を設ける
- ログインユーザーが公開付箋にコメントを投稿できる
- コメントは投稿者名・投稿日時・本文をスレッド形式で一覧表示する
- 自分のコメントは削除できる（他人のコメントは削除不可）
- コメントは付箋削除時にCASCADE削除される
- コメント投稿・削除はAJAXで即時反映する

#### APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| POST | `/api/memo/<id>/comment` | コメントを投稿 |
| DELETE | `/api/comment/<id>` | コメントを削除（投稿者のみ） |

### 12. 貢献スコア・ランキング（新規追加）

コルクボードへの参加度を可視化してSNS的な動機付けを与える。

#### スコア計算

| アクション | 得点 |
|-----------|------|
| 公開付箋を追加 | +3点 |
| 他ユーザーにいいねされる | +2点／いいね |
| コメントを投稿 | +1点 |

#### 表示

- コルクボード画面のツールバーに「ランキング」ボタンを設ける
- クリックでそのフォルダ内の貢献スコアランキングをモーダルで表示する
- 1位には 🥇、2位には 🥈、3位には 🥉 のアイコンを表示する
- 自分のスコアをハイライト表示する

#### APIエンドポイント

| メソッド | パス | 説明 |
|----------|------|------|
| GET | `/api/folder/<id>/ranking` | フォルダ内の貢献スコアランキングを返す |

## 非機能要件・共通仕様

- 付箋の座標はドラッグ終了時にAJAX（fetch API）でサーバーへ保存する
- Markdown本文のXSSはレンダリング時に `marked.js` のサニタイズ機能で対処する
- フォルダ削除・非公開メモの操作は権限チェックをサーバーサイドで実施する（URLの直打ちも含む）
- 描画・いいね・コメントの削除権限はサーバーサイドで検証する
